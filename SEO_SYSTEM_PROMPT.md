# Промпт: SEO-платформа с дашбордом для рекрутингового сайта

## Контекст

У нас есть лендинг рекрутингового агентства (работа за границей: ОАЭ, Саудовская Аравия, Катар, Кувейт). Два направления: бьюти-сфера и сфера сервиса. Текущий сайт — статичный React + Vite + Tailwind. Нужно перевести его на динамическую платформу с полной SEO-системой и админ-дашбордом.

Исходный код сайта находится в `C:\Users\Admin\Documents\website\`.

---

## Задача

Построить современную SEO-платформу со следующими возможностями:

### 1. Динамический сайт (перевод со static React)

**Стек:**
- **Next.js 14+ (App Router)** — SSR/SSG для SEO, серверные компоненты
- **Tailwind CSS v4** — стили (уже используется)
- **PostgreSQL** (Supabase или Neon) — хранение SEO-данных, контента, метрик
- **Prisma** — ORM
- **NextAuth.js** — авторизация для дашборда
- **Деплой:** Vercel (или VPS с nginx, если указано)

**Требования к сайту:**
- Перенести все текущие секции (Hero, Beauty, Service, Trust, Process, FAQ, Urgency, Quiz, Footer, WhatsApp)
- Контент страниц должен тянуться из базы данных (CMS-подобная система)
- SSR для всех страниц (критично для SEO)
- Динамические meta-теги (title, description, OG-tags) из базы данных
- Автоматическая генерация sitemap.xml и robots.txt
- Поддержка мультиязычности (RU, EN — на будущее)
- Структурированные данные (JSON-LD микроразметка)

---

### 2. SEO-модуль (автоматизация)

#### 2.1. Внутренняя/техническая SEO-оптимизация

Создать систему, которая:

**Метаданные (управляемые через дашборд):**
- Title и description для каждой страницы — редактируемые через дашборд
- H1–H6 заголовки — структура управляемая
- Canonical URLs — автоматическая генерация, с возможностью ручной перезаписи
- Open Graph теги (og:title, og:description, og:image) — для каждой страницы
- Twitter Card теги
- Hreflang теги (для мультиязычности)

**Индексация:**
- Автоматическая генерация sitemap.xml при изменении контента
- robots.txt с управлением через дашборд
- Мета-тег robots (index/noindex, follow/nofollow) для каждой страницы
- Автопинг Google Search Console при обновлении sitemap

**Микроразметка (JSON-LD):**
- Organization
- WebSite + SearchAction
- BreadcrumbList
- FAQPage (для секции FAQ)
- JobPosting (для вакансий)
- Review/AggregateRating (для отзывов)
- HowTo (для процесса трудоустройства)

**Техническое здоровье:**
- Автоматическая проверка битых ссылок (cron, раз в неделю)
- Проверка скорости загрузки через Google PageSpeed Insights API
- Мониторинг Core Web Vitals
- Проверка мобильной адаптивности
- Логирование ошибок 404 и редиректов

#### 2.2. Семантическое ядро

Создать модуль для работы с ключевыми словами:

**Сбор и анализ:**
- Интеграция с Google Keyword Planner API (или SerpAPI/DataForSEO как альтернатива) для получения объёма запросов
- Хранение ключевых слов в базе: ключевое слово, категория, группа, объём запросов, конкуренция, тренд
- Привязка ключевых слов к конкретным страницам сайта
- Целевые локации: ОАЭ, Саудовская Аравия, Катар, Кувейт + СНГ (источник трафика)

**Трекинг позиций:**
- Автоматический съём позиций по ключевым словам (через SerpAPI или аналог)
- Хранение истории позиций (ежедневно/еженедельно)
- Графики динамики позиций по времени
- Сравнение с конкурентами (до 5 доменов)

**Рекомендации:**
- Автоматические рекомендации: какие страницы оптимизировать, какие ключевые слова добавить
- Анализ контент-гэпов (ключевые слова конкурентов, которых нет у нас)

#### 2.3. Автоматические задачи (cron)

| Задача | Частота |
|--------|---------|
| Съём позиций по ключевым словам | Раз в неделю |
| Проверка битых ссылок | Раз в неделю |
| Аудит скорости (PageSpeed) | Раз в неделю |
| Обновление объёма запросов ключевых слов | Раз в 3 месяца |
| Обновление трендов ключевых слов | Раз в 3 месяца |
| Генерация SEO-отчёта | Раз в месяц |
| Проверка технического здоровья | 3 раза в неделю |
| Обновление sitemap.xml | При изменении контента |

### 3. Админ-дашборд

**Авторизация:**
- Логин/пароль (NextAuth.js)
- Роли: admin (полный доступ), editor (только контент)

**Разделы дашборда:**

#### 3.1. Главная (Overview)
- Общий трафик (Google Analytics / Search Console)
- Текущие позиции по топ-10 ключевым словам
- Здоровье сайта (score)
- Core Web Vitals
- Последние ошибки/предупреждения

#### 3.2. Страницы (Content Management)
- Список всех страниц сайта
- Для каждой страницы: редактирование title, description, OG-tags, контента
- Предпросмотр SERP-сниппета (как будет выглядеть в Google)
- Чеклист SEO-оптимизации для каждой страницы (длина title, наличие ключевых слов, alt у картинок)

#### 3.3. Ключевые слова
- Таблица всех ключевых слов: слово, объём, конкуренция, текущая позиция, привязанная страница
- Фильтры по категории, группе, позиции
- Добавление/удаление ключевых слов
- Импорт из CSV
- Графики трендов

#### 3.4. Позиции
- График позиций по времени (line chart)
- Таблица: ключевое слово → текущая позиция → изменение за неделю/месяц
- Фильтр по странице, категории

#### 3.5. Технический аудит
- Последний скан: битые ссылки, ошибки, предупреждения
- PageSpeed Score (mobile + desktop)
- Core Web Vitals (LCP, FID, CLS)
- Рекомендации по исправлению

#### 3.6. Отчёты
- Ежемесячный SEO-отчёт (автогенерация)
- Содержание: трафик, позиции, здоровье, выполненные задачи, рекомендации
- Экспорт в PDF
- История отчётов

#### 3.7. Настройки
- robots.txt (редактирование)
- Домены конкурентов для сравнения
- API-ключи (Google Search Console, PageSpeed, SerpAPI)
- Расписание cron-задач
- Уведомления (email при падении позиций, ошибках)

### 4. Интеграции (API)

| Сервис | Зачем | API |
|--------|-------|-----|
| Google Search Console | Трафик, клики, позиции | Search Console API |
| Google Analytics 4 | Трафик, поведение | GA4 Data API |
| Google PageSpeed Insights | Скорость, Core Web Vitals | PageSpeed API |
| SerpAPI / DataForSEO | Позиции в выдаче, ключевые слова | REST API |

**Важно:** Если API-ключ не настроен — модуль не падает, а показывает заглушку "Настройте интеграцию".

### 6. Схема базы данных (основные таблицы)

```prisma
model Page {
  id          String   @id @default(cuid())
  slug        String   @unique
  title       String
  metaTitle   String?
  metaDesc    String?
  ogTitle     String?
  ogDesc      String?
  ogImage     String?
  content     Json     // блоки контента
  robots      String   @default("index, follow")
  canonical   String?
  isPublished Boolean  @default(true)
  keywords    Keyword[]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model Keyword {
  id           String   @id @default(cuid())
  word         String
  category     String?
  group        String?
  volume       Int?     // объём запросов
  competition  Float?   // 0-1
  trend        Json?    // массив значений по месяцам
  pageId       String?
  page         Page?    @relation(fields: [pageId], references: [id])
  positions    Position[]
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model Position {
  id        String   @id @default(cuid())
  keywordId String
  keyword   Keyword  @relation(fields: [keywordId], references: [id])
  position  Int
  url       String?
  location  String   // "UAE", "KW", "QA", "SA"
  date      DateTime @default(now())
}

model AuditLog {
  id        String   @id @default(cuid())
  type      String   // "broken_link", "pagespeed", "health"
  status    String   // "ok", "warning", "error"
  data      Json
  createdAt DateTime @default(now())
}

model Report {
  id        String   @id @default(cuid())
  month     DateTime
  data      Json     // структурированный отчёт
  pdfUrl    String?
  createdAt DateTime @default(now())
}

model SeoSetting {
  id    String @id @default(cuid())
  key   String @unique
  value String
}
```

### 8. UX-требования к дашборду

- Минималистичный дизайн (как Vercel Dashboard / Linear)
- Sidebar навигация
- Тёмная/светлая тема
- Мобильная адаптация (но десктоп приоритет)
- Графики: Recharts или Chart.js
- Таблицы: сортировка, фильтрация, пагинация
- Уведомления: toast-сообщения
- Загрузка данных: skeleton-лоадеры

---

### 9. Важные замечания

- **Не ломать существующий лендинг** — он должен продолжать работать как раньше, просто теперь на Next.js
- **Graceful degradation** — если API-интеграция не настроена, система работает без неё
- **Безопасность** — дашборд доступен только авторизованным пользователям
- **Не использовать формулировку «работа для девушек»** на сайте (бизнес-требование)
- Заменить `YOUR_NUMBER` / `YOUR_USERNAME` на реальные контакты (спросить у пользователя)
- **WhatsApp номер и Telegram** — должны быть настраиваемые через дашборд (раздел настройки)

### 10. Автоматическое обновление данных (раз в 3 месяца)

Система должна автоматически:
1. Пересобирать объёмы запросов по всем ключевым словам через Google Keyword Planner / SerpAPI
2. Обновлять тренды по категориям за последние 48 месяцев
3. Пересчитывать уровень конкуренции
4. Генерировать рекомендации по обновлению структуры сайта
5. Создавать квартальный SEO-отчёт с рекомендациями
6. Отправлять уведомление админу о завершении обновления

Эти задачи запускаются cron-ом или могут быть запущены вручную через дашборд (кнопка "Обновить данные").
